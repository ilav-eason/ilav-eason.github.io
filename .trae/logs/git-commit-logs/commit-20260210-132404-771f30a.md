## 错误和警告

### 错误: Git 无法索引 nul 文件
- **上下文:** `error: short read while indexing nul` 和 `fatal: adding files failed`
- **原因:** nul 是 Windows 系统保留的设备名称，Git 无法正确处理
- **解决方案:**
  1. 使用 `git update-index --force-remove nul` 从索引中移除
  2. 在 `.gitignore` 中添加 `nul` 防止再次被跟踪
  3. 在 `_config.yml` 的 exclude 中添加 `nul`
- **状态:** 已解决

### 错误: GitHub Actions 构建失败
- **上下文:** `The process '/opt/hostedtoolcache/Ruby/3.1.6/x64/bin/gem' failed with exit code 1`
- **原因:** GitHub Actions 使用 Ruby 3.1，但本地环境使用 Ruby 3.2，Gemfile.lock 不匹配
- **解决方案:** 将 `.github/workflows/jekyll.yml` 中的 `ruby-version: '3.1'` 改为 `ruby-version: '3.2'`
- **状态:** 已解决

### 错误: Git 推送失败
- **上下文:** `Failed to connect to github.com port 443 after 21083 ms: Couldn't connect to server`
- **原因:** 网络连接问题，需要通过代理访问 GitHub
- **解决方案:** 配置 Git 代理
  ```bash
  git config --global http.proxy http://127.0.0.1:7890
  git config --global https.proxy https://127.0.0.1:7890
  ```
- **状态:** 已解决

### 警告: PowerShell 不支持 &&
- **上下文:** 使用 `&&` 连接命令时报错 `标记"&&"不是此版本中的有效语句分隔符`
- **解决方案:** 分开执行命令或使用分号
- **状态:** 已解决
